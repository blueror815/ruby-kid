JSON parameter name Class property Data Type
id categoryId NSInteger

gender_group NSInteger - either 'M', 'F', or 'MF'
male_icon_url NSString
male_camera_background_url NSString
male_icon_background_color NSString - hex value like "#ffccff"
male_hides_name Boolean - whether should hide the category name along with the icon
female_icon_url NSString
female_camera_background_url NSString
female_icon_background_color NSString - hex value like "#ffccff"
female_hides_name Boolean - whether should hide the category name along with the icon

Negotiation info between the users: items, comments, status flags.

id NSInteger - Trade ID

comments { "user": User hash, "comment": String }
waiting_for_counter_offer Boolean in Integer - 1 or 0
waiting_for_meeting_place Boolean in Integer - 1 or 0
waiting_for_parent_approval Boolean in Integer - 1 or 0
waiting_for_user_id Integer - the user who should respond to action next.
last_meeting_place_set_by Integer - the user who last set meeting place.
finished Boolean in Integer - 1 or 0
cancelled Boolean in Integer - 1 or 0
sent_completed_notification Boolean in Integer - 1 or 0
completion_confirmed Boolean in Integer - 1 or 0

Model: Notification

Message from another system with action type like “Likes Your Item” or “Invite to Trade”.
JSON parameter name Class property Data Type
id notificationID NSInteger
title title NSString
tip tip NSString
related_model_type relatedModelType NSString - values like Item, User, Child, Parent, Trade, ItemComment, FavoriteItem, OfferResponse
related_model_id relatedModelID NSInteger - the record ID of related_model_type
status status NSString - values like WAIT, VIEWED
created_at createdAt NSString in date-time format yyyy-MM-dd'T'HH:mm:ssZ
priority NSInteger - higher than 0 if user needs to pay more attention.
starred Boolean - whether this message type is important and given a star
sender sender NSDictionary - User hash of the sender

Item Info API Requests

/items/{id}.json (Request method: GET)

Detailed item information query.
Request Parameter Required? Description
id Y id integrated into the URL is required.
JSON Response Parameter Data Type Description
item Hash of Item JSON attributes
user Hash of User JSON attributes
is_in_cart Boolean in Integer form
is_in_favorite_items Boolean in Integer form Whether signed-in user set Like this item.
is_follower Boolean in Integer form Whether signed-in user is following the item's seller shop
permission Integer Flag representing the combination of permissions the signed-user has to access the item.
comment_threads Array of <NSDictionary of "from_user": buyer User hash, "comments": <Array of ItemComment hash> > Lists of comments based on buyers
other_items Array of Item JSON The seller's other active items.
favorite_item_ids Array of Integer Whether the signed-in user likes some of the other items.

/items/likes.json or /items/favorites.json (Request method: GET)

Fetches the user's selected Like/Favorite items.
Request Parameter Required? Description
JSON Response Parameter Data Type Description
Hash of Item Search Result

Item Approval API Requests

/inventory/manager.json (Request method: GET)

Retrieval the list of items posted in various status. If signed-in user is a child, the list would be only child's items. Else if user is a parent, the children's items would be included.
Request Parameter Required? Description
tab N Values: active | pending | all, Default: active
JSON Response Parameter Data Type Description
/ Array of Item JSON attributes Root of the response hash is an array.

/inventory/activate.json (Request method: GET, PUT or POST)

Approve or re-activate item(s). Required parameter is either id or item_ids
Request Parameter Required? Description
id N One single item
item_ids N Multiple item IDs (Integer)
decline_item_ids N To decline other items at the same, set this Array of item IDs (Integer).
item_1234256 N In the pattern of "item_123456" where 123456 is item ID, set to update item with this Item attributes
JSON Response Parameter Data Type Description
success Boolean in Integer form (0 | 1)
item_count Integer Number of items updated

/inventory/deactivate.json (Request method: GET, PUT or POST)

End item(s). Required parameter is either id or item_ids
Request Parameter Required? Description
id N One single item
item_ids N Multiple items
JSON Response Parameter Data Type Description
success Boolean in Integer form (0 | 1)

/inventory/decline.json (Request method: GET, PUT or POST)

Decline the pending item(s). Required parameter is either id or item_ids
Request Parameter Required? Description
id N One single item
item_ids N Multiple items
JSON Response Parameter Data Type Description
success Boolean in Integer form (0 | 1)
